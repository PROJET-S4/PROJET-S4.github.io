I) Réflexions autour du projet

A) Pourquoi ce choix ?

  • Volonté d'avoir un jeu différent des standards. .

  • Pas vraiment d'intérêt pour les plateformer "classiques".

  • Choix de créer un shoot-them-up rétro-futuriste (Inspiré de Space Invader).

B) Comment avons nous procédé ?

  • Premier Brainstorming pour trouver un type de jeu.

  • Un second pour le titre (créant alors "Chronicles of Naruberaria").

IV) Problèmes rencontrés et solutions proposées

A) Concernant les API

  • No 1 => Problème de liaison entre Angular et Laravel (Unknown error dès le remplissage d'un formulaire).

  • => Correction via un changement de valeur pour la variable "apiUrl" du fichier environement.ts

  • No 2 => Problème de validation du formulaire d'inscription (redirection au lieu d'inscription)

  • => Modification du <'Input'> qui contenait à la base un routerLink="/game"

  • No 3 => Problème de droits (scope error) et de validation pour la modification du profil utilisateur.

  • => Modification des rôles contenus dans les routes du fichier api.php (sous laravel) et modification du type d'une variable (de String vers Int).

B) Concernant le jeu

  • No 1 => Les ennemis tués disparaissaient mais l'entité existait encore ils n'étaient "pas vraiment morts".

  • => Correction par reprogrammation complète des ennemis les transformant en groupe de sprite (ce qui les rend plus simple à retirer).
